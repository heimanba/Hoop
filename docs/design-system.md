# Hoop 设计系统

## 品牌与产品原则

- **成长陪伴**：每个页面都优先表达持续进步，而不是纯竞技压力。
- **真实训练感**：保留运动产品的节奏、任务感和明确反馈，不做幼态化表达。
- **首页先任务**：首页第一屏优先放今日打卡、训练入口和最近分析，趋势统计降为二级。
- **AI 像教练**：分析结果聚焦动作重点、下一步建议、进步对比，不堆复杂技术术语。
- **儿童场景清晰安全**：状态反馈简短、明确、不过度夸张；危险操作必须显式确认。

## Token 规范

### 颜色

- `brand` / `brandEmphasis`：主品牌橙，承载主按钮、重要 CTA、核心激励状态。
- `training` / `game` / `duel` / `ai` / `achievement` / `parent`：业务场景标签色。
- `success` / `warning` / `error` / `info` / `disabled`：统一语义反馈色。
- `background` / `surface` / `surfaceRaised` / `surfaceMuted` / `border` / `textPrimary` / `textSecondary` / `textTertiary`：统一浅色/深色语义层级。

规则：

- 页面不得直接硬编码 `Color(...)`。
- 业务色不能替代错误态。
- AI 强调色只用于分析相关模块，不占据首页第一视觉中心。

### 字体

- `display`：首页成长摘要和强情绪标题。
- `title1` / `title2` / `title3`：导航、区块、卡片标题。
- `body` / `bodyEmphasis` / `callout`：正文、说明、重点文案。
- `caption` / `captionEmphasis`：标签、状态、时间、辅助文案。
- `metricLarge` / `metricMedium`：统计数值与趋势卡。

规则：

- 只使用圆角系统字系。
- 数字指标优先使用 metric 字体，不与普通正文混用。

### 间距与尺寸

- 基础步进：`4 / 8 / 12 / 16 / 20 / 24 / 32`
- 页面左右边距：`20`
- 卡片内边距：`16`
- 紧凑卡片内边距：`12`
- 模块间距：`24`
- 控件高度：`52`
- 标准列表行高：`56`

规则：

- 新页面默认使用 `pageMargin` 和 `sectionGap`。
- 不允许随意新增独立 spacing 常量，除非是跨页面复用的新 token。

### 圆角、描边、阴影、动效

- 圆角统一不超过 `8pt`
- 描边使用 `thin` 或 `emphasis`
- 卡片阴影保持轻量，仅用于抬升主要内容
- 动效仅使用 `quick` / `standard` / `emphasis`

规则：

- 不做大幅弹跳动画。
- 不做悬浮玻璃卡片叠卡效果。

## 组件规范

### 按钮

- `PrimaryButtonStyle`：主行动，如开始训练、完成上传、确认提交。
- `SecondaryButtonStyle`：次级行动，如上传视频、查看更多、切换模式。
- 危险操作使用系统 destructive 角色，不自己发明危险色按钮。

### 卡片

- 统一使用 `HoopCard`
- 卡片内保持“标题 + 描述 + 状态/操作”结构
- 不允许在卡片内再套装饰性卡片

### 标签与状态

- 统一使用 `StatusBadge`
- 待完成优先 `warning`
- 已完成优先 `success`
- AI 相关优先 `ai`
- 比赛/单挑使用各自业务色

### 指标与分析

- 指标卡统一使用 `MetricTile`
- AI 分析摘要统一使用 `AIInsightCard`
- AI 页面优先用“问题 - 建议 - 对比”三段式表达

## 页面模板

### 首页 / 今日打卡流

- 顶部：成长摘要、连续打卡、阶段感受
- 主区：今日任务、训练入口、最近视频分析
- 次区：最近比赛/单挑、周进步、推荐动作

### 训练页

- 顶部训练计划
- 中部动作清单
- 视频上传入口固定为显著次级操作

### 比赛 / 单挑页

- 最近事件
- 表现摘要
- 关键瞬间与复盘建议

### AI 分析页

- 最近分析结论
- 本周观察重点
- 历史对比入口

### 我的 / 家长视角页

- 账号信息
- 周期趋势、成长里程碑、勋章
- 账号权限与退出登录

## SwiftUI 落地规则

- 通过 `Theme` 注入到 app root，页面用 `@Environment(Theme.self)` 读取主题。
- 页面中不得散落硬编码字号、颜色、圆角、padding。
- 公共视觉结构优先使用小型组件与 `ViewModifier`，避免新建样式型 view model。
- 新增页面时优先从五类页面模板出发，而不是从空白画布重做布局。
